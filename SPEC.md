# WBTI 项目规格说明

## 一、项目概述

### 1.1 产品定位

**WBTI（Workplace「疯格」Type Indicator）** 是一款基于「职场」的**趣味人格测试**产品。用户完成 **16 道选择题**后，获得一个专属的**「职场疯格」标签**及配套的**精神分析报告**（文案向、娱乐向，非临床心理测评）。

### 1.2 核心价值


| 价值维度     | 说明                        |
| -------- | ------------------------- |
| **情绪出口** | 用幽默解构职场压力，提供低负担的情绪宣泄与自嘲场景 |
| **社交货币** | 结果页适合截图/分享，满足自我表达与传播      |
| **身份认同** | 标签化归类，强化「不只我一个人这样」的共鸣     |


### 1.3 目标用户


| 画像          | 特征                          |
| ----------- | --------------------------- |
| **打工人**     | 18–40 岁，有职场经历或正在工作          |
| **互联网语境用户** | 熟悉 MBTI / SBTI 等梗文化，接受自黑与玩梗 |
| **社媒分享者**   | 习惯在朋友圈、小红书、微博等分享趣味测试结果      |


### 1.4 竞品对标


| 产品       | 定位              | WBTI 差异化              |
| -------- | --------------- | --------------------- |
| MBTI     | 偏专业心理人格框架       | 更轻量、更娱乐、更本土化职场语境      |
| SBTI     | 解构版 MBTI，「屎」文化梗 | **聚焦职场场景**，场景更集中      |
| 网易云性格色彩等 | 偏情感向测试          | 更偏**吐槽与发疯文学**，语气更「班味」 |

## 二、产品范围与版本边界

### 2.1 MVP 范围（建议）

- **单线流程**：欢迎/说明 → 16 题 → 结果页（标签 + 报告）。
- **无账号体系**（可选：本地保存最近一次结果）。
- **分享**：结果页支持系统分享 / 复制链接 / 保存图片（具体依平台能力定）。

### 2.2 明确非目标（首期）

- 不提供临床、招聘、晋升等**严肃决策依据**。
- 不做复杂题库后台 CMS；客户端可为扩大可玩性维护**静态大题库**与平衡抽题规则。

## 三、功能需求规格

### 3.1 测验流程


| ID   | 需求             | 验收要点                            |
| ---- | -------------- | ------------------------------- |
| F-01 | 题目共 16 道，均为选择题 | 每题必选才可进入下一题；可定义是否允许返回修改         |
| F-02 | 进度可见           | 显示当前题号/总题数或等价进度文案（可含娱乐化表述）       |
| F-03 | 提交后生成结果        | 根据计分/规则映射到唯一或有限集合中的「疯格」类型       |
| F-04 | 结果页展示          | **类型名（标签）** + **报告正文**（多段，由内容定） |
| F-05 | 重新测试           | 可一键清空或重新开始；新一局重洗题序/选项序；大题库须**平衡抽题**（见下节）；本地计数等行为须文档化 |

### 3.1.1 大题库抽题与「人格覆盖」平衡算法

**目标**：从规模可扩展的静态题库（当前 **320** 条，可增）中，每局抽取 **n=16** 道
**不重复**题目；在随机性的同时，保证三轴（a/b/c，对应八种疯格的三维符号）在
题组上的**覆盖强度均匀**，避免某一轴样本过少导致分型漂移。

**内容与配置载体**：母题、变体文案表、接梗矩阵以 **`data/*.json`** 维护；**八种结果**
正文在 **`data/result-catalog.json`**（**`reportSections`** 四段、八种格共用一套小标题）；
进度条与分享文案在 **`data/ui-copy.json`**；
视觉 token（`--*` 变量等）在 **`data/theme.json`**（运行时
注入 `html`）；产品可调参数（一局题数、变体行数、平局符号、情境默认、**选项接梗行**
`showOptionQuipDefault`、**成就 storage 与阈值**）在 **`data/app-config.json`**，由
`js/appConfig.js` 校验导出；加载与结构说明见
仓库 **`data/README.md`**。

**`balanceAxis`（覆盖槽）**

- 每条题目必须带 **`balanceAxis ∈ {a,b,c}`**，表示该题计入「哪一轴覆盖桶」。
- 入库规范：由内容作者指定；未指定时实现上可回退为 **按 `patternId % 3` 轮转**
  分配到 a、b、c（保证长期扩充时三桶可均衡供给）。
- 说明：覆盖槽与选项上的 `scores` 独立；**分型仍由全部选项分数求和**决定，
  `balanceAxis` **不参与加分**，仅用于抽题平衡。

**平衡判定（与 n 成比例）**

- 记 `lo = ⌊n/3⌋`，`hi = ⌈n/3⌉`。当且仅当 a、b、c 三桶题数均落在 `[lo, hi]`
  内且总和为 n 时，视为**满足人格覆盖**（n=16 时唯一整数分为 **5,5,6** 的排列）。

**抽题流程（实现：`pickBalancedSessionFromBank`）**

1. **均匀随机无放回**：从题库中随机取 n 道不同 `id` 的题目（**同一套内 `id`
   不得重复**；替换题时亦禁止选已入套的 `id`；返回前可断言校验）。**母题 cap**：
   同一 `patternId`（题干正文相同）在每局出现次数 ≤ `⌈n / 母题种数⌉`（当前 n=16、
   16 母题 → 每母题至多 **1** 次），一局题干互不重复。`variantContext.scene`
   仍入库；**界面默认不展示** 情境标签，可在 `createQuizView` 传
   `showVariantScene: true` 开启。
2. 若人格覆盖与母题 cap 均已满足 → 跳到步骤 4；否则进入 **局部替换**：
   - 记过多桶 `over = argmax(计数)`，过少桶 `under = argmin(计数)`。
   - 在已选题中随机选一道 `balanceAxis === over` 的题，从题库剩余题中优先选
     `balanceAxis === under` 的题替换之；若无，则选「非 over」的题替换。
   - 重复有限次（默认约数百次）直至人格覆盖与母题 cap 均满足或无法改善。
3. 若多轮 **全量重试**（默认约百次）仍不满足 → **贪心兜底**：按 a/b/c 三池
   各取 `lo` 道，再将余下的 `n - 3·lo` 道随机分配到若干桶（每池题量须 ≥ 目标），
   最后整体洗牌（兜底路径仍须满足母题 cap）。
4. **题序与选项序**：将得到的 n 题再经 `buildQuizSession` 洗题序与各题内选项序
   （不改变选项与分数绑定）。

**题库约束**：每种 `balanceAxis` 在库中至少要有 `lo` 道题；若将来 n 增大，须
同步保证三池容量。母题按 `patternId % 3` 落入 a/b/c 时，当前 **6 / 5 / 5** 道母题
各在一池；`n=16` 时人格覆盖需 **5,5,6**，多出的 **1** 题只能落在仍有「未用母题
槽位」的轴（实现：贪心兜底按 `patternRepeatCap` 可抽取上限分配 remainder，避免
把第 6 题硬塞给仅有 5 道母题的桶）。

### 3.2 结果与算法

- **输入**：每题选项对应分值或维度向量（例如：隐忍 vs 爆发、内耗 vs 外耗等；维度名由内容
团队定）。
- **输出**：映射到 **N 种结果类型之一**（N 建议在规格中写死，如 8 / 12 / 16）。
- **平局规则**：同分时优先级表或次要维度，必须在文档中写清，保证可测试。

### 3.3 分享与传播

- 结果页在常见移动端宽度下**截图友好**（安全区、关键文案不被裁切）。
- **OG 图 / 分享文案**（若做 Web）：标题、描述、缩略图规范单独列出。

### 3.4 可访问与体验（建议最低线）

- 键盘/读屏可操作（Web 时）。
- 色弱友好：不只靠颜色区分状态。

## 四、内容与文案规格

- **语气**：幽默、自嘲、职场梗；避免侮辱特定群体、疾病污名化、性别/地域歧视。
- **免责声明**：显著位置说明「仅供娱乐」；**不用于招聘/选拔**；不构成心理或
  医学诊断或职业决策依据。结果页固定区与分享/复制文案须含等价表述。
- **题库**：每题题干 + 2–4 个选项；可维护**静态大题库**与平衡抽题；选项长度
  上限（避免移动端换行灾难）可在规格写死。

## 六、成功指标（示例）
